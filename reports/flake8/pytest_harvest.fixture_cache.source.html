19 """
20 Decorates a fixture so that it is saved in `store`. `store` can be a dict-like variable or a string
21 representing a fixture name to a dict-like session-scoped fixture. By default it uses the global 'fixture_store'
22 fixture provided by this plugin.
23  
24 After executing all tests, `<store>` will contain a new item under key `<key>` (default is the name of the fixture).
25 This item is a dictionary <test_id>: <fixture_value> for each test node where the fixture was setup.
26  
27 ```python
28 import pytest
29 from pytest_harvest import saved_fixture
30  
31 @pytest.fixture
32 @saved_fixture
33 def dummy():
34 return 1
35  
36 def test_dummy(dummy):
37 pass
38  
39 def test_synthesis(fixture_store):
40 print(fixture_store['dummy'])
41 ```
42  
43 Note that for session-scoped and module-scoped fixtures, not all test ids will appear in the store - only those
44 for which the fixture was (re)created.
45  
46 Users can save additional views created from the fixture instance by applying transforms (callable functions). To
47 do this, users can provide a dictionary under the `views` argument, containing a `{<key>: <procedure>}` dict-like.
48 For each entry, `<procedure>` will be applied on the fixture instance, and the result will be stored under `<key>`.
49 `save_raw` controls whether the fixture instance should still be saved in this case (default: `True` if
50 `views is None`, `False` otherwise).
51  
52 :param store: a dict-like object or a fixture name corresponding to a dict-like object. in this dictionary, a new
53 entry will be added for the fixture. This entry will contain a dictionary <test_id>: <fixture_value> for each
54 test node. By default fixtures are stored in the `fixture_store``fixture.
55 :param key: the name associated with the stored fixture in the store. By default this is the fixture name.
56 :param views: an optional dictionary that can be provided to store views created from the fixture, rather than (or
57 in addition to, if `save_raw=True`) the fixture itself. The dict should contain a `{<key>: <procedure>}`
58 dict-like. For each entry, `<procedure>` will be applied on the fixture instance, and the result will be stored
59 under `<key>`.
60 :param save_raw: controls whether the fixture instance should be saved. `None` (Default) is an automatic behaviour
61 meaning "`True` if `views is None`, `False` otherwise".
62 :return: a fixture that will be stored
63 """
